Integration

Sitesearch exposes the API https://api.sitesearch.svolve.com/search for all your search queries.

As we want to keep the integration to very simple and intuitive, a simple GET as shown below would suffice. Please note that since you are charged based on the number of search requests, ensure that you dont share the website token with anyone.

    &siteId=
        ]]>
Query Param Name Mandatory Description
q Yes Search string with properly URL encoded.
siteId Yes Token of the website.
lang No Language filter to apply on the results. Defaults to search documents across all language
from No Start Index in the search results. To be used with pagination of results.
size No Number of search results returned. To be used with pagination of results along with from
highlight No Provides highlight for the matching text. Defaults to true.
suggest No Provides suggestions for spell check corrections. Defaults to true.

As the query string can contain special chars, ensure that its properly URL encoded before the request is submitted to the API. You can use the from and size query params for pagination of results.

The sitesearch index is backed by elastic search. So the response structure is completely same as how elastich search would return. Below is an example of it.
